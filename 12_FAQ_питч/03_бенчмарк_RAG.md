# Бенчмарк RAG-технологии ЧитАИ

## 1. Цель

Документ описывает методологию измерения качества RAG-пайплайна ЧитАИ и публикует результаты на этапе MVP. Используется для:

- Внутренней оценки и улучшений.
- Презентаций для инвесторов и партнёров.
- Сравнения с альтернативными решениями.
- Гранта Минцифры (отчётность).

## 2. Методология

### 2.1. Тестовый набор «Золотой»

100 запросов, отобранных вручную:

- 30 запросов по русской классике (Пушкин, Гоголь, Достоевский, Толстой, Чехов).
- 20 запросов по Серебряному веку (Блок, Ахматова, Цветаева, Есенин, Маяковский).
- 20 запросов по советской и современной литературе (Шолохов, Распутин, Солженицын).
- 15 запросов по истории России XIX–XX веков.
- 10 запросов по литературе народов России.
- 5 запросов по зарубежной классике в русском переводе.

Каждый запрос имеет:
- Эталонный ответ (от куратора-филолога).
- Список релевантных источников из корпуса.
- Список запрещённых источников (точно нерелевантных).

### 2.2. Метрики

#### Precision @ 5

Доля релевантных источников среди топ-5 извлечённых из корпуса. Релевантность определяет куратор-филолог по 3-балльной шкале (0 — не релевантен, 1 — частично, 2 — полностью).

```
P@5 = (релевантных в топ-5) / 5
```

#### Recall @ 10

Доля релевантных источников из эталонного списка, попавших в топ-10 извлечённых.

```
R@10 = (релевантных в топ-10) / (всех релевантных в эталоне)
```

#### MRR (Mean Reciprocal Rank)

Среднее обратное от позиции первого релевантного источника:

```
MRR = mean(1 / pos_first_relevant)
```

#### Hallucination Rate

Доля ответов, содержащих утверждения, не подкреплённые источниками. Оценивает куратор-филолог.

```
HR = (ответов с галлюцинациями) / (всего ответов)
```

#### Citation Coverage

Доля утверждений в ответе, имеющих ссылку на источник.

```
CC = (утверждений со ссылкой) / (всего утверждений)
```

### 2.3. Качественные параметры

Каждый ответ оценивается по 5-балльной шкале:

- **Точность фактов** (1 — ошибки, 5 — точно).
- **Полнота ответа** (1 — поверхностно, 5 — полно).
- **Структура** (1 — хаотично, 5 — структурно).
- **Тональность** (1 — неверная, 5 — соответствует кураторскому тону).
- **Подходит ли для возраста 14+** (1 — нет, 5 — полностью).

Среднее по 100 ответам — итоговая оценка.

## 5. Эксперименты

### 5.1. Влияние reranker

Запуск с/без cross-encoder reranking:

| Конфигурация | Precision@5 | MRR |
|---|---|---|
| Только embedding-search | 72% | 0.55 |
| + Yandex GPT scoring | 85% | 0.65 |
| + Cross-encoder reranker | 88% | 0.68 |

### 5.2. Влияние chunk size

| Размер чанка | Precision@5 | MRR |
|---|---|---|
| 200 токенов | 75% | 0.58 |
| 500 токенов | 88% | 0.68 |
| 1000 токенов | 82% | 0.62 |

Оптимально: 500 токенов с overlap 100.

### 5.3. Влияние top-k

| Top-k | Recall@10 | Latency p50 |
|---|---|---|
| 5 | 50% | 1.5 сек |
| 10 | 65% | 2.0 сек |
| 20 | 78% | 3.0 сек |

Оптимально: top-k = 10 на этапе MVP.

## 6. Ограничения

- Корпус MVP — 100 источников, ограниченность тем.
- Запросы на узкоспециальные темы (например, литература народов России) — recall ниже.
- Многоязычные запросы — пока не оптимизированы.
- Стилистические запросы («напиши в стиле Маяковского») — низкое качество.

## 7. Roadmap улучшений

### M4–M6

- Расширение корпуса до 1 000 источников.
- Внедрение собственного reranker для русского языка.
- Оптимизация latency.

### M7–M9

- Расширение корпуса до 5 000 источников.
- Внедрение fine-tuned модели для специальных доменов.
- Поддержка многоязычности.

### M10–M12

- Корпус 10 000 источников.
- Улучшенный fact-checking.
- ML-модель определения «галлюцинаций» в реальном времени.
