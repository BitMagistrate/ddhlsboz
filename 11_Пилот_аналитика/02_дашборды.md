# Дашборды ЧитАИ

## 1. Уровни дашбордов

| Уровень | Аудитория | Платформа |
|---|---|---|
| Внутренний продукт | Команда ЧитАИ | Yandex DataLens |
| Кабинет партнёра | Школа / библиотека / музей | Встроен в Web UI |
| Дашборд региона | Региональное министерство | Встроен в Web UI или DataLens |
| Финансовый | CEO + инвесторы | Yandex DataLens |
| Маркетинговый | Команда маркетинга | Yandex DataLens + Yandex Metrika |

## 2. Внутренний продуктовый дашборд

### 2.1. Активность

- DAU, MAU.
- Новые пользователи в день.
- Удержание (cohort анализ).
- Сессии в день.
- Средняя длительность сессии.

### 2.2. Контент

- Маршруты создано / завершено.
- Топ-10 запрошенных тем.
- Топ-10 источников.
- Среднее число шагов в маршруте.
- Время на маршрут.

### 2.3. Тренажёр

- Заданий выполнено.
- Средний балл.
- Топ-10 тем.
- Динамика по уровням (ОГЭ vs ЕГЭ).

### 2.4. RAG-метрики

- Precision (по бенчмарку).
- Recall.
- Hallucination rate.
- Latency p50, p95.
- Доля ответов «не нашёл».

### 2.5. Технические

- Uptime сервиса.
- Error rate API.
- Длина очереди.
- Использование CPU/RAM/Disk.
- Стоимость API LLM в день.

## 3. Кабинет партнёра — школа

### 3.1. Сценарии уроков

- Создано в спринт.
- Использовано в классах.
- Топ-10 тем.

### 3.2. Активность учеников

- Активных учеников.
- Сессии в неделю.
- Маршруты завершено.
- Задания тренажёра выполнено.
- Средний балл по классу.

### 3.3. Прогресс по темам

- Средний прогресс по программе.
- Слабые темы класса.
- Сильные темы класса.

### 3.4. Рекомендации

- Что доработать в сценариях.
- На что обратить внимание учеников.

## 4. Кабинет партнёра — библиотека

### 4.1. Виджет

- Запросов в день / неделю.
- Маршрутов сформировано.
- Конверсия маршрут → визит библиотеки.
- Время до визита.
- Доля повторных посетителей.

### 4.2. Тематика запросов

- Топ-20 тем.
- Возрастная структура.
- Сезонность.

### 4.3. Аудитория

- Доля молодёжи 14–22 в посетителях (динамика по неделям).
- Соотношение онлайн / офлайн запросов.

### 4.4. Сравнение

- С аналогичными библиотеками региона (в обезличенном виде).

## 5. Кабинет партнёра — музей

### 5.1. Включения в маршруты

- Маршрутов с музеем в день / неделю.
- Топ-5 тем, по которым музей рекомендуется.

### 5.2. Конверсия

- Маршрут → визит.
- Время до визита.
- Доля «Пушкинской карты» в посещаемости.

### 5.3. Контент-генератор

- Постов сгенерировано в неделю.
- Опубликовано (с пометкой партнёра).
- Engagement в социальных сетях (если предоставлены данные).

### 5.4. Виртуальные туры

- Запусков в неделю.
- Доля онлайн-визита перешедших в офлайн-визит.

## 6. Дашборд региона (для министерств)

### 6.1. Общая активность

- Активные пользователи в регионе (DAU, MAU).
- Динамика по месяцам.
- Возрастная структура.
- Гендерная структура (с согласия пользователей).

### 6.2. Учреждения культуры

- Подключено к платформе: библиотек, музеев, школ.
- Запросов через виджеты в день.
- Конверсия маршрут → визит.

### 6.3. «Пушкинская карта»

- Транзакций по карте в регионе через ЧитАИ-маршруты.
- Динамика по неделям.
- Топ-10 событий по доле от ЧитАИ-маршрутов.

### 6.4. Тематические интересы

- Топ-30 тем.
- Динамика по сезонам.
- Возрастная структура каждой темы.

### 6.5. ОГЭ/ЕГЭ

- Учеников в тренажёре.
- Средний балл по предметам (литература, история).
- Сравнение с прошлым годом.

### 6.6. Соответствие нацпроекту

- Доля посещений учреждений культуры от 14–22 (целевой показатель нацпроекта).
- Динамика.
- Сравнение с другими регионами.

### 6.7. Экспорт

- В Excel / PDF.
- Готовые отчёты для совещаний министерства.
- API для интеграции с системой министерства.

## 7. Финансовый дашборд (CEO + инвесторы)

### 7.1. Выручка

- MRR (monthly recurring revenue).
- ARR (annual recurring revenue).
- ARR по сегментам (B2C, B2B, B2G).
- Средний чек.
- Динамика по месяцам.

### 7.2. Затраты

- Burn rate.
- Расходы по статьям.
- Стоимость API LLM.
- CAC (стоимость привлечения).

### 7.3. Юнит-экономика

- LTV.
- LTV/CAC.
- Pay-back period.
- ARPU.

### 7.4. Прогнозы

- Прогноз выручки на 12 мес.
- Прогноз runway (сколько месяцев до конца кэша).
- Сценарий «оптимистичный/реалистичный/пессимистичный».

## 8. Маркетинговый дашборд

### 8.1. Каналы трафика

- Источники (utm_source).
- Конверсия по источнику.
- CAC по источнику.

### 8.2. Кампании

- Активные кампании.
- Бюджет освоен.
- Регистраций / конверсий.
- ROI кампаний.

### 8.3. Контент

- Просмотры блога.
- Подписчики Telegram / VK / Email.
- ER (engagement rate).
- Прослушивания подкаста.

### 8.4. PR

- Упоминания в СМИ.
- Тон упоминаний (положительный / нейтральный / отрицательный).
- Покрытие аудитории.

## 9. Технические аспекты

### 9.1. Платформа

- Yandex DataLens — основной инструмент визуализации.
- Источник данных — PostgreSQL (через прокси / read-replica).
- Для регионального дашборда — встраивание в Web UI ЧитАИ.

### 9.2. Обновление

- Real-time для критичных метрик (DAU, активные пользователи).
- Раз в час — для большинства метрик.
- Раз в сутки — для финансовых.

### 9.3. Доступ

- Внутренние дашборды — команда + инвесторы.
- Кабинеты партнёров — только этот партнёр.
- Дашборд региона — региональное министерство и согласованные сотрудники.

### 9.4. Безопасность

- Аутентификация через JWT.
- Логирование действий.
- Скрытые показатели (например, поимённый список — только агрегаты).

## 10. Принципы

- Один экран = один вопрос. Не нагружать дашборд избыточной информацией.
- Числа сопровождаются динамикой.
- Цветовая палитра нейтральная (см. `04_Бренд/02_цвета_типографика.md`).
- Мобильная адаптация для ключевых дашбордов.
- Экспорт в PDF/Excel — обязательная функция.

## 11. Аналитический отчёт партнёру (раз в квартал)

Формат: PDF на 8–12 страниц.

### Структура

1. Резюме (1 страница).
2. Активность.
3. Контент.
4. Конверсии.
5. Сравнение с целевыми KPI.
6. Качественная обратная связь.
7. Рекомендации.
8. План на следующий квартал.

Подготовка: автоматизирована, ручная корректировка ответственным менеджером пилота.

## 12. Соответствие 152-ФЗ

- Все ПД на дашбордах — обезличены.
- Нет возможности идентифицировать конкретного пользователя.
- Доступ к dashboards журналируется.
- Аудит по запросу регулятора — готовность.
