# Синтетические данные для демо и тестирования ЧитАИ

## 1. Назначение

Документ описывает синтетические данные, используемые в демо-прототипе ЧитАИ, в тестах и в презентациях. Это нужно для:

- Демо без реальных пользователей (на этапе MVP).
- Юнит-тестов без зависимости от реальной БД.
- Презентаций для инвесторов / партнёров до накопления реальных кейсов.

## 2. Принципы

- Все синтетические данные явно маркированы как синтетические в публикациях.
- Объёмы и распределения подобраны реалистично, на основе исследований.
- Имена пользователей — выдуманные (с пометкой «вымышленный персонаж»).
- Для презентаций кейсов — открыто указано «синтетический пример».

## 4. Скрипт генерации

`backend/scripts/generate_synthetic_data.py` (на MVP):

```python
import random
from datetime import datetime, timedelta

def generate_dau(month_index, base=800, growth=1.2):
    return int(base * (growth ** month_index))

def generate_topics(n=100):
    weights = [14, 9, 8, 7, 6, 5, 4, 4, 4, 3, 3, 3, 3, 2, 25]
    topics = ["Серебряный век", "Война и мир", "Преступление и наказание",
              "Подготовка к ЕГЭ литература", "Подготовка к ЕГЭ история",
              "Военная проза", "Гоголь", "Чехов", "Краеведение",
              "Литература народов России", "История XX века",
              "Современная проза", "Зарубежная классика",
              "Советская поэзия (доступная)", "Прочее"]
    return random.choices(topics, weights=weights, k=n)
```

(Полный скрипт — в `backend/scripts/`.)

## 5. Использование

### 5.1. В демо-прототипе

- Дашборд региона показывает синтетические данные.
- Кабинет учителя показывает синтетический класс.
- Тренажёр показывает синтетические задания (но с реальными критериями ФИПИ).

### 5.2. В презентациях

- Слайды с цифрами: используем реальные данные пилотов (когда есть) или явно помечаем «оценочные / синтетические».
- Кейсы школ / библиотек: до пилотов — синтетические, после — реальные с разрешения.

### 5.3. В тестах

- Юнит-тесты не зависят от реальных пользователей.
- Используется фиксированный seed для воспроизводимости.
- Корпус источников — public domain (всегда реальные).

## 6. Маркировка

В публикациях, презентациях и интерфейсе:

- «Синтетический пример» — крупно, в верхней части.
- «Данные сгенерированы для демонстрации» — в подвале.
- В дашборде — баннер «Демо-режим».

## 7. После пилотов

После накопления реальных данных:

- Замена синтетических на реальные (с разрешения партнёров).
- Сохранение структуры синтетических данных как базы для тестов.
- Регулярное обновление по реальным трендам.

## 8. Этический контроль

- Не использовать синтетические данные для введения в заблуждение.
- Не выдавать синтетические данные за реальные.
- При вопросе журналиста или эксперта о происхождении цифр — открыто признаваться, если синтетические.
- При публикации после пилотов — чёткое разграничение «синтетический пример» vs «реальный кейс».
