# Реестр операций обработки персональных данных ЧитАИ

Реестр ведётся в соответствии с 152-ФЗ. Обновляется при добавлении новых процессов и сервисов. Хранится в защищённом виде.

## Реестр операций

### Операция 1. Регистрация и ведение учётной записи Пользователя

| Параметр | Значение |
|---|---|
| Цель | Предоставление доступа к Сервису |
| Правовое основание | Согласие, договор оферты |
| Категории субъектов | Пользователи Сервиса (физ. лица 14+) |
| Состав ПД | имя/псевдоним, e-mail, Telegram ID, возрастная группа, IP, cookie, идентификатор сессии |
| Действия | сбор, запись, систематизация, накопление, хранение, обезличивание, удаление |
| Источники | сам субъект |
| Передача третьим лицам | нет (за исключением хостинга — Yandex Cloud) |
| Срок хранения | 12 месяцев после удаления учётной записи |
| Меры защиты | TLS 1.3, AES-256, 2FA для сотрудников, журналирование, разграничение доступа |
| Ответственный | DPO |

### Операция 2. Формирование маршрута чтения

| Параметр | Значение |
|---|---|
| Цель | Персонализация ответов и рекомендаций |
| Правовое основание | Согласие, договор оферты |
| Категории субъектов | Пользователи Сервиса |
| Состав ПД | запросы пользователя, история ответов, отметки like/dislike, предпочтения |
| Действия | сбор, запись, использование, обезличивание |
| Передача | нет |
| Срок хранения | 12 месяцев в персонализированном виде, далее обезличенный |
| Меры защиты | стандартные |

### Операция 3. Кабинет учителя (B2B)

| Параметр | Значение |
|---|---|
| Цель | Подготовка сценариев уроков, аналитика по классу |
| Правовое основание | Договор с образовательной организацией, согласие законных представителей учеников |
| Категории субъектов | Учителя, учащиеся 14+ (с согласия законного представителя — 14–18) |
| Состав ПД | имя/псевдоним учителя, e-mail, школа, класс, агрегированные ответы учащихся (без личных идентификаторов учащихся в выгрузках третьим лицам) |
| Действия | сбор, запись, использование, обезличивание |
| Передача | нет |
| Срок хранения | 12 месяцев после прекращения договора с образовательной организацией |
| Меры защиты | стандартные + раздельные роли «учитель» и «ученик» |

### Операция 4. Кабинет библиотекаря (B2B)

| Параметр | Значение |
|---|---|
| Цель | Аналитика тематических интересов, виджеты на сайт библиотеки |
| Правовое основание | Договор с библиотекой |
| Категории субъектов | Сотрудники библиотеки |
| Состав ПД | имя, e-mail, должность, библиотека |
| Действия | стандартные |
| Передача | нет |
| Срок хранения | срок договора |
| Меры защиты | стандартные |

### Операция 5. Дашборд региона (B2G)

| Параметр | Значение |
|---|---|
| Цель | Аналитика тематических интересов молодёжи в регионе для министерства |
| Правовое основание | Договор с региональным министерством |
| Категории субъектов | Сотрудники министерства |
| Состав ПД | имя, должность, e-mail, ведомство |
| Действия | стандартные |
| Передача | нет |
| Срок хранения | срок договора |
| Меры защиты | стандартные + разграничение по правам доступа к разрезам данных |

Важно: в дашборде региона показываются только обезличенные агрегированные данные пользователей. Пользовательские профили в министерство не передаются.

### Операция 6. Маркетинговая рассылка

| Параметр | Значение |
|---|---|
| Цель | Информирование пользователей и заказчиков |
| Правовое основание | Отдельное согласие |
| Категории субъектов | Пользователи и заказчики, давшие согласие |
| Состав ПД | имя, e-mail |
| Действия | сбор, использование, передача провайдеру рассылок (на территории РФ) |
| Передача | оператор рассылок (российский провайдер на договоре) |
| Срок хранения | до отзыва согласия |
| Меры защиты | стандартные + механизм отписки в каждом письме |

### Операция 7. Платёжные операции

| Параметр | Значение |
|---|---|
| Цель | Прием оплаты подписок |
| Правовое основание | Договор оферты |
| Категории субъектов | Пользователи, оформившие подписку |
| Состав ПД | имя, e-mail; банковские данные обрабатываются исключительно платёжным провайдером (PCI DSS) |
| Действия | передача платёжному провайдеру |
| Передача | сертифицированный российский платёжный провайдер |
| Срок хранения | в соответствии с законодательством о бухгалтерском учёте (5 лет) |
| Меры защиты | без хранения банковских данных на стороне Оператора |

## Уведомление в Роскомнадзор

После регистрации ООО Оператор подаёт уведомление об обработке персональных данных в Роскомнадзор по форме, утверждённой приказом, с приложением реестра.

## Аудит и пересмотр

- Реестр пересматривается раз в полугодие.
- Аудит соответствия — раз в год силами внешнего консультанта.
- При добавлении нового процесса — обновление в течение 30 дней.
