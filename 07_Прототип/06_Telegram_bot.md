# Telegram-бот ЧитАИ

## Назначение

Telegram-бот `@ChitaiBot` — основной канал доступа к сервису для аудитории молодёжи 14–22. Реализует функции Куратора, Тренажёра ЕГЭ и личного кабинета пользователя.

## 1. Команды

| Команда | Назначение |
|---|---|
| `/start` | Начало работы, регистрация, выбор возрастного режима |
| `/route` | Построить новый маршрут чтения |
| `/quiz` | Перейти к тренажёру ЕГЭ/ОГЭ |
| `/sources` | Посмотреть список источников по теме |
| `/settings` | Настройки (имя, возраст, интересы, согласия) |
| `/feedback` | Оставить обратную связь |
| `/help` | Помощь и контакты |

## 2. Сценарии

### Сценарий 1. Первый запуск

1. Пользователь: `/start`.
2. Бот:
   > Здравствуйте. Я ЧитАИ — российский ИИ-куратор маршрутов чтения по фондам РГБ и НЭБ. Помогу подобрать книги, лекции, музеи под ваши интересы. Как к вам обращаться?
3. Пользователь вводит имя или псевдоним.
4. Бот:
   > Подтвердите ваш возраст: 14–17 / 18+. Это нужно для применения возрастных фильтров.
5. Пользователь выбирает.
6. Бот:
   > Прочитайте Политику обработки ПД и Условия использования. Согласны?
7. Пользователь подтверждает.
8. Бот:
   > Готово. Что вас интересует? Например: «Серебряный век», «Подготовка к ЕГЭ по литературе», «Военная проза», или введите свободный запрос.

### Сценарий 2. Запрос маршрута

1. Пользователь: «Хочу разобраться в Достоевском».
2. Бот вызывает `POST /api/curator/route` с темой.
3. Бот возвращает маршрут на 4 недели:
   > Маршрут «Достоевский: 4 недели».
   > Неделя 1: «Преступление и наказание» (главы 1–3) + лекция «Раскольников и идея».
   > Неделя 2: «Преступление и наказание» (главы 4–6).
   > …
4. Кнопки: «начать», «упростить», «другая тема».

### Сценарий 3. Тренажёр ЕГЭ

1. Пользователь: `/quiz`.
2. Бот: «Литература или история?».
3. Пользователь выбирает «литература».
4. Бот: «ОГЭ или ЕГЭ?».
5. Пользователь выбирает «ЕГЭ».
6. Бот выдаёт задание из `GET /api/quiz/items?subject=literature&level=ege`.
7. Пользователь отвечает.
8. Бот вызывает `POST /api/quiz/submit`, возвращает балл и обратную связь.

### Сценарий 4. Запрос источников

1. Пользователь: «Что есть по Серебряному веку».
2. Бот вызывает `GET /api/sources?q=Серебряный век`.
3. Бот возвращает 5–10 источников с авторами, названиями, ссылками.

### Сценарий 5. Обратная связь

1. Пользователь: `/feedback`.
2. Бот: «Опишите проблему или предложение».
3. Пользователь вводит текст.
4. Бот вызывает `POST /api/feedback`.
5. Бот: «Спасибо. Мы получили обратную связь».

## 3. Архитектура

- Long polling в MVP, webhook в продакшене.
- aiogram 3, FSM для многошаговых диалогов.
- Хранение состояния — в Redis (продакшен) / в памяти (MVP).
- Вызовы backend — через `httpx`.

## 4. Конфигурация

Файл `.env`:

```
TELEGRAM_BOT_TOKEN=...
BACKEND_URL=https://api.chitai.ru
LOG_LEVEL=INFO
```

## 5. Безопасность

- Не хранит сообщения пользователей в Telegram-инфраструктуре дольше необходимого.
- При получении подозрительного запроса (попытка инжекции, запрещённые темы) — отказ с нейтральным сообщением.
- 2FA для административного доступа к боту (BotFather + Telegram).

## 6. Возрастные режимы

- При запуске бот спрашивает возраст.
- В режиме 14+ применяются дополнительные фильтры (см. `05_Юр_пакет/08_этический_протокол.md`).
- Изменение возрастного режима — через `/settings`.

## 7. Метрики

- Daily Active Users (DAU).
- Retention 7-day.
- Среднее число сообщений на пользователя в неделю.
- Конверсия в маршрут (доля пользователей, начавших и завершивших маршрут).
- Конверсия в тренажёр (доля, использовавшая тренажёр).

## 8. Отказоустойчивость

- При недоступности backend — бот возвращает дежурное сообщение «Извините, временно недоступно. Попробуйте через несколько минут».
- Логи — в Yandex Cloud Logging.
- Алертинг — на ошибки API.

## 9. Локализация

- Русский язык.
- Расширения: татарский, башкирский, якутский (M9+, по запросу регионов).

## 10. Интеграции

- Yandex SpeechKit — для голосовых сообщений (бот может озвучивать ответы).
- Внешние сервисы для билетов («Кассир.ру», «Афиша») — в M6+.
- «Пушкинская карта» — интеграция через open API (если предоставит «Госуслуги» или ЦБ).

## 11. MVP-имплементация

В MVP-прототипе (этот репозиторий):

- Бот реализован как заглушка в `backend/bot.py` (или отдельная папка `bot/`).
- Команды `/start`, `/route`, `/help`, `/feedback` — работают на детерминированных сценариях.
- При наличии `TELEGRAM_BOT_TOKEN` в окружении — бот может быть запущен локально.
- Для демо-целей — интерфейс работы с Куратором доступен также через сайт и мокап.

## 12. Развёртывание в продакшене

- Webhook на `https://bot.chitai.ru/webhook`.
- Запуск в отдельном Pod в Kubernetes.
- Health-check каждые 30 секунд.
- Auto-restart при падении.
- Логи и метрики — в общем стеке (Loki + Prometheus).
