# Стратегия тестирования ЧитАИ

## 1. Уровни тестирования

### 1.1. Unit-тесты

- Backend: pytest, покрытие ≥ 80%.
- Frontend: vitest + React Testing Library, покрытие ≥ 70%.
- Тестируются: бизнес-логика, валидация, RAG-компоненты, утилиты.

### 1.2. Интеграционные тесты

- Backend: pytest + httpx для API-эндпоинтов.
- Тестируются: эндпоинты, взаимодействие с БД (через тестовую БД), RAG-pipeline целиком.

### 1.3. E2E-тесты

- Playwright для Web UI.
- aiogram-test для Telegram-бота.
- Сценарии: регистрация, маршрут, тренажёр, обратная связь.

### 1.4. Нагрузочное тестирование

- Locust / k6.
- Целевая нагрузка MVP: 100 одновременных пользователей.
- Целевая нагрузка продакшен: 10 000 одновременных пользователей.

### 1.5. Бенчмарк качества

- «Золотой набор» из 200 вопросов.
- Прогон еженедельно.
- Метрики: precision, recall, hallucination rate, latency.

### 1.6. Безопасность

- bandit (Python).
- npm audit (JS).
- pip-audit.
- Регулярный pen-test (раз в полугодие).

### 1.7. Доступность

- axe-core (автоматизированно).
- Ручное тестирование на NVDA, VoiceOver.
- WCAG 2.1 AA — обязательное соответствие.

## 2. Test data

- Синтетические данные пользователей (без реальных ПД).
- Синтетические данные дашборда (генерируются скриптом).
- Корпус источников — public domain (общественное достояние).
- Тестовые промпты — от команды и эксперта-филолога.

## 3. CI

В каждом PR:

- Lint.
- Unit-тесты.
- Интеграционные тесты.
- Сборка.
- При merge в `main` — также E2E.

В nightly:

- Бенчмарк «золотой набор».
- Нагрузочное тестирование.
- Аудит зависимостей.

## 4. Acceptance-критерии релиза

- Все автоматизированные тесты пройдены.
- Бенчмарк не регрессировал > 5%.
- Покрытие тестами не упало > 5%.
- Smoke-тесты в staging пройдены.
- QA-ревью пройдено.

## 5. Bug-tracking

- Задачи: GitHub Issues / Yandex Tracker.
- Приоритеты: P0 (блокер), P1 (критичный), P2 (важный), P3 (косметика).
- SLA: P0 — 4 часа, P1 — 24 часа, P2 — 5 дней, P3 — спринт.

## 6. Регрессионное тестирование

- Регрессионный набор автоматизирован.
- Прогон перед каждым релизом.
- Ручное регрессионное тестирование критичных сценариев — ежемесячно.

## 7. Тестирование на разных устройствах

- Desktop: Chrome, Firefox, Edge, Safari.
- Mobile: iOS Safari (iPhone 12+), Android Chrome (Android 9+).
- Tablet: iPad, Android планшеты.
- Telegram: iOS, Android, Web.

## 8. Тестирование возрастных режимов

- 14+ режим: 50 запросов на чувствительные темы — проверка фильтрации.
- 18+ режим: те же 50 запросов — проверка корректности.
- Тестовые сценарии — поддерживаются автоматизированно.

## 9. Метрики тестирования

- Покрытие unit-тестами.
- Покрытие интеграционными тестами.
- Время выполнения CI-pipeline.
- Доля автоматизированных регрессионных тестов.
- Среднее число багов на релиз.
- Время от обнаружения до фикса (MTTR).

## 10. Тестирование RAG

### 10.1. Точность ссылок

- Каждый ответ должен содержать ссылку на источник.
- Ссылка должна быть валидной (HTTP 200).
- Цитата из источника должна совпадать с контекстом.

### 10.2. Защита от галлюцинаций

- Тестовый набор «провокационных» запросов (ошибочные данные, выдуманные имена).
- Ожидаемое поведение: «не нашёл данных в фондах».

### 10.3. Кросс-моделирование

- Прогон одного и того же запроса через YandexGPT и GigaChat.
- Сравнение ответов.
- При значимом расхождении — флаг для ручного ревью.

## 11. Тестирование 152-ФЗ

- Аудит хранения ПД.
- Проверка локализации.
- Проверка маскирования в логах.
- Проверка экспорта/удаления данных по запросу.

## 12. Команда тестирования

- QA-инженер: 1 человек на старте, 2 к M9.
- Ручное тестирование: команда + 1 внешний QA на регрессию.
- Эксперт-филолог: ревью качества литературных ответов.

## 13. Чек-лист QA

- [ ] Все эндпоинты возвращают корректные коды статуса.
- [ ] Все формы валидируют ввод.
- [ ] Возрастные режимы работают.
- [ ] Источники и ссылки корректны.
- [ ] Бот реагирует на все команды.
- [ ] Дашборд показывает корректные метрики.
- [ ] Тренажёр оценивает по критериям ФИПИ.
- [ ] Сайт открывается на мобильных.
- [ ] Доступность WCAG 2.1 AA.
- [ ] Нет утечек ПД в логах.
