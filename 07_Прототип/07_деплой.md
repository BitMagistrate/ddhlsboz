# Деплой ЧитАИ

## 1. Среды

| Среда | URL | Назначение |
|---|---|---|
| local | http://localhost:5173 (frontend), http://localhost:8000 (backend) | Локальная разработка |
| staging | staging.chitai.ru | Тестовая среда для команды и пилотов |
| production | chitai.ru, api.chitai.ru | Прод |
| MVP | Fly.io URL (backend), Vercel/Netlify (frontend) | Демо-прототип для Нейрофеста |

## 2. MVP-деплой (этот репозиторий)

### Backend на Fly.io

1. Установить flyctl (используется автоматически в Devin).
2. Команда деплоя: `<deploy_backend dir="backend"/>` (без persistent volume для MVP).
3. Получить публичный URL (например, `https://chitai-api.fly.dev`).

### Frontend на Vercel/Netlify

1. Сборка: `npm run build` в директории `frontend/`.
2. Команда деплоя: `<deploy_frontend dir="frontend/dist"/>`.
3. Получить публичный URL (например, `https://chitai-frontend.netlify.app`).
4. В `.env` фронтенда указан backend URL.

### HTML-мокап

1. Файл `mockup/index.html` — самодостаточный.
2. Можно открыть локально или развернуть как статику.

## 3. Production-деплой

### Backend в Yandex Cloud Kubernetes

- Базовая конфигурация: 3 ноды (vCPU 8, RAM 32 GB).
- Автоскейлинг по CPU > 70%.
- PostgreSQL: Yandex Managed PostgreSQL, Multi-AZ.
- Redis: Yandex Managed Redis.
- Объектное хранилище: Yandex Object Storage.
- KMS: Yandex KMS для ключей.

### Frontend на Yandex Cloud

- Build на стороне CI (GitHub Actions / Yandex DevOps).
- Размещение в S3-совместимом хранилище + CDN.
- HTTPS: Yandex Certificate Manager.
- Кэширование статики: 1 год.

### Telegram-бот

- Отдельный Pod в кластере K8s.
- Webhook URL: `https://bot.chitai.ru/webhook`.
- Long-polling fallback.

## 4. CI/CD

### GitHub Actions

- На push в `main`:
  1. Lint (ruff, eslint).
  2. Тесты (pytest, vitest).
  3. Сборка Docker-образов.
  4. Push в Yandex Container Registry.
  5. Деплой в staging.
- На тег `v*.*.*`:
  1. Деплой в production (после ручного approval).

```

```

## 6. Секреты

- Хранение: Yandex Lockbox (продакшен) / GitHub Secrets (CI).
- Ротация ключей: раз в год, при увольнении сотрудника — немедленно.
- Доступ: only-as-needed.

## 7. Мониторинг

- Prometheus + Grafana.
- Алерты: на error rate > 1%, latency p95 > 3 сек, доступность < 99%.
- Логирование: Loki (структурированное).
- Трассировка: OpenTelemetry → Jaeger.

## 8. Backup и DR

- PostgreSQL: ежедневные backup + WAL-архивирование.
- Хранение: 30 дней.
- DR-план: восстановление из backup в SberCloud за 1 час.
- Проверка backup: раз в квартал.

## 9. Безопасность

- WAF на входе (Yandex Cloud).
- DDoS-защита (Yandex Smart Protection).
- TLS 1.3.
- Регулярный pen-test (раз в полугодие).
- Sec-аудит (раз в год).

## 10. Стоимость инфраструктуры (оценочно)

- Yandex Cloud Kubernetes: ~80 000 ₽/мес (3 ноды).
- Managed PostgreSQL: ~30 000 ₽/мес (Multi-AZ).
- Object Storage: ~5 000 ₽/мес.
- Yandex GPT 5 Pro: тарификация по токенам, оценка ~50 000 ₽/мес при 10 000 активных пользователей.
- CDN: ~5 000 ₽/мес.
- Прочее (мониторинг, KMS, Audit Trails): ~10 000 ₽/мес.

Итого: ~180 000 ₽/мес на старте, рост по мере увеличения нагрузки.

## 12. Rollback

- При регрессии: rollback в Kubernetes до предыдущей версии (revision-1).
- Время rollback: ≤ 5 минут.
- Уведомление команды и пользователей.
- Анализ причин в течение 24 часов.
