# ЧитАИ — полное представление проекта

> Версия документа: 1.0 · Дата: 09.05.2026 · Состояние кода: PR #1+#2+#3+#4 в `main`, CI зелёный, 181 тест.

## 0. Одной строкой

**ЧитАИ — российский ИИ-куратор индивидуальных культурно-образовательных маршрутов по фондам РГБ и НЭБ для молодёжи 14–22 лет (через Пушкинскую карту), учителей литературы и истории, региональных библиотек, музеев и Минкультуры/Минпросвещения.**

Полностью на российском стеке (YandexGPT 5 Pro + GigaChat MAX, Yandex SpeechKit, FRIDA-эмбеддинги, Yandex Cloud), без OpenAI/Anthropic/Google.

## 1. Проблема

### 1.1 Что не так сегодня

| Аудитория | Проблема | Подтверждение |
|---|---|---|
| Школьники 14–18 | Школьная программа по литературе устарела, мотивация падает, ЕГЭ-задания — стандартизированные. ИИ-помощники типа `zachet.app` поощряют «обмани учителя»: минимизировать «ИИ-Score» в готовом сочинении. | Анализ конкурентов; стилистика лендингов («ИИ-проверка не палится»). |
| Студенты 18–22 | Пушкинская карта: 5 000 ₽ / год на культурные мероприятия — не знают, на что её потратить. 60% карт не активированы. | Открытые данные Минкультуры. |
| Учителя | Готовят персональные траектории вручную (~6 ч / класс / неделя). Нет инструмента, который даёт точную цитату с подтверждением источника. | Глубинные интервью (`11_Пилот/`). |
| Региональные библиотеки/музеи | Низкий охват подростков (≤8% молодёжи 14–22 посещают библиотеки регулярно). Нет канала продвижения мероприятий именно той аудитории, у которой есть Пушкинская карта. | Статистика Минкультуры РФ. |
| Минкультуры / Минпросвещения | Нет инструмента, который связывает фонды РГБ/НЭБ + Пушкинскую карту + школьную программу + ЕГЭ в один маршрут с измеримыми метриками вовлечённости. | Открытые цели нацпроектов «Культура» и «Образование». |

### 1.2 Почему ИИ-генератор «как у zachet.app» — неправильный путь

**Идеология zachet.app:** «генерируй сочинение → пройди ИИ-проверку учителя». Это:
- Этически проблемно (пособничество в недостоверной успеваемости).
- Юридически уязвимо (152-ФЗ, авторское право, протокол этики Минпросвещения).
- Стратегически бесперспективно для B2G — ни одно министерство не подпишет контракт с инструментом, который позиционируется как «обмани учителя».

**Идеология ЧитАИ:** «прочитай книгу → пойми смысл → подтверди цитатой → закрепи через интервальное повторение → найди мероприятие в Пушкинской карте». Это работает с РФ-нормативкой (ФГОС, ФИПИ, нацпроект «Культура»), а не против неё.

## 2. Решение — продукт ЧитАИ

### 2.1 Ценностное предложение в одной фразе

> «Прочитай в 4 раза больше, понимай в 10 раз глубже — без зубрёжки, с подтверждением каждой цитаты ссылкой на public-domain издание.»

### 2.2 Пользовательские сценарии

**Сценарий 1 — Школьник 16 лет («Капитанская дочка», ЕГЭ)**
1. Открывает Telegram-бота `@ChitaiBot` или PWA на смартфоне.
2. Пишет: «помоги подготовиться к ЕГЭ по «Капитанской дочке»».
3. Получает 4-недельный маршрут: каждую неделю 1–2 главы + 5 ключевых цитат (с привязкой к public-domain изданию РГБ) + 3 темы для эссе + 1 квиз.
4. Каждая цитата проверяема: клик → открывается фрагмент в фондах РГБ/НЭБ.
5. Тренажёр: SRS-флешкарточки (алгоритм SM-2), повторения по интервалам 1 / 3 / 7 / 14 / 30 дней.
6. Бонус: подборка мероприятий из Пушкинской карты по теме (литературные вечера, экскурсии, спектакли) в его регионе.

**Сценарий 2 — Учитель литературы**
1. Заходит в B2B-кабинет («Teacher Review»).
2. Загружает ученическое сочинение в `.docx`.
3. ИИ подсвечивает спорные места (фактические ошибки, слабую аргументацию, отсутствие цитат) и предлагает корректные источники из РГБ.
4. Импортирует целый класс одной кнопкой → получает 30 индивидуальных маршрутов на неделю.

**Сценарий 3 — Региональная библиотека (Татарстан, Ярославль)**
1. Подписывается на B2G-тариф.
2. Получает дашборд: какие книги читают подростки в её регионе, какие мероприятия Пушкинской карты «вытягивают» аудиторию.
3. Через 30 дней — отчёт ROI: «вложили X ₽, привели Y подростков на мероприятия, увеличили оборот фондов на Z%».

### 2.3 Доказуемая дифференциация

| Параметр | ЧитАИ | zachet.app | ChatGPT-обёртки |
|---|---|---|---|
| Каждый ответ — с цитатой и ссылкой на public-domain | ✅ | ❌ | ❌ |
| RAG поверх фондов РГБ/НЭБ (а не общий веб) | ✅ | ❌ | ❌ |
| Пушкинская карта интегрирована | ✅ | ❌ | ❌ |
| Соответствие 152-ФЗ (consent / export / forget) | ✅ | ❌ | ❌ |
| Полностью российский стек (YandexGPT, GigaChat, Yandex SpeechKit) | ✅ | смешанный | OpenAI / Anthropic |

### 3.2 Стек (полностью российский)

**LLM**
- **Primary**: GigaChat MAX (Сбер) — `GigaChat-Max:2.0.28.2`, OAuth 2.0 (две стадии: токен → chat completions).
- **Secondary**: YandexGPT 5 Pro — `gpt://<folder>/yandexgpt/latest`, API-Key auth.
- **Fallback**: Mock с детерминированным выводом (для CI/тестов и graceful degradation).
- Контракт ответа: Pydantic v2 `CuratorRouteResponse` (все поля валидируются перед отправкой клиенту).
- Тайм-аут: 8 секунд на каждый LLM-вызов.
- **OpenAI / Anthropic / Google не используются** — стратегическое решение для B2G.

**Эмбеддинги & ASR/TTS**
- Эмбеддинги: YandexGPT Embeddings (для production) + FRIDA / multilingual-e5 (для локальных тестов).
- TTS: **Yandex SpeechKit** (озвучка маршрута, кнопка «🔊 Озвучить» в Кураторе).
- ASR: Yandex SpeechKit (в roadmap, PR #6+).
- Иллюстрации (для маркетинга): Kandinsky 3.1.

**Backend**
- FastAPI 0.110, Python 3.11, Pydantic v2.
- async/asyncio, httpx (для всех внешних вызовов с retry + connection pool).
- 38 HTTP-эндпоинтов.
- Гибридный поиск: BM25 + dense + Reciprocal Rank Fusion.
- Stemmer для русской морфологии (`backend/app/search/stemmer.py`).
- 145 тестов (pytest + respx для HTTP-моков).

**Frontend**
- React 18 + TypeScript + Vite.
- Tailwind CSS + shadcn/ui (компоненты).
- recharts (графики дашборда), lucide-react (иконки).
- PWA: manifest.webmanifest, иконки 192/512, sw.js (precache shell + network-first /api + SWR для статики).
- WCAG: skip-link, role=tablist/tab, aria-selected/-controls, roving tabindex, focus-visible.
- 27 тестов (vitest + @testing-library/react).

**Telegram-бот**
- aiogram 3.x, httpx.
- Меню: Куратор / Тренажёр / Информация.
- Покрывает основные сценарии PWA для пользователей без браузера / в школьных сетях.
- 9 тестов (pytest + respx).

**База данных**
- PostgreSQL 15 + pgvector (production).
- Redis (кэш сессий, OAuth-токены GigaChat).
- В демо-режиме — in-memory + mock-корпус.

**Инфраструктура**
- Yandex Cloud (Compute, Object Storage для public-domain корпуса, Managed PostgreSQL, KMS для секретов).
- Резервная площадка: SberCloud.
- Observability: Prometheus `/metrics` (RPS, p95-latency, LLM cost, hallucination rate), JSON-логи, OpenTelemetry traces.

**CI/CD**
- GitHub Actions, 3 джобы (backend / frontend / bot).
- Backend: `ruff check` + `pytest -q --tb=short` (Python 3.11, Poetry).
- Frontend: `npm run lint` + `tsc -b && vite build`.
- Bot: `ruff check . || true` + `pytest -q` (uv для управления зависимостями).
- Всё на каждый push в любую ветку.

## 4. Текущая функциональность (что уже работает)

### 4.1 Реализовано (в `main`, CI зелёный)

#### 4.1.1 Куратор (`/api/curator/*`)
- **Маршрут на 4 недели** с разбиением по неделям. Каждая неделя содержит фрагмент произведения, цитату с указанием стр./главы, ссылку на public-domain издание РГБ.
- **JSON-Schema контракт** ответа (Pydantic v2 `CuratorRouteResponse`): обязательные поля `query / summary / weeks / sources / disclaimer / llm_provider / llm_model`, валидация перед отправкой клиенту.
- **Реальный GigaChat MAX** (live-проверен: «Патриотизм и милосердие» — две главные темы «Капитанской дочки», 45 токенов, провайдер `gigachat`, модель `GigaChat-Max:2.0.28.2`).
- **Fallback-роутер**: `gigachat → yandex → mock` (graceful degradation, никогда не падает на 500 из-за провайдера).
- **Тайм-аут 8 секунд** на каждый LLM-вызов.
- **Экспорт маршрута**: `.md` (Markdown) и `.ics` (iCalendar — добавить чтения в Google Calendar / Yandex Calendar).
- **TTS озвучка**: кнопка «🔊 Озвучить» — Yandex SpeechKit, MP3 через `<audio controls>`.
- **Интеллект-карта** (`/api/curator/mindmap`): nodes (query / author / book / theme) + edges (relationships). UI — списочное представление; React Flow граф — в roadmap.

#### 4.1.2 Гибридный поиск (`/api/corpus/*`)
- BM25 + dense embeddings + RRF reranker.
- Корпус public-domain: ~30 произведений (Пушкин, Лермонтов, Гоголь, Толстой, Достоевский и др.) с метаданными ЕГЭ, school_grade, ege_topics.
- Морфология: stemmer для русских окончаний.
- Бенчмарк harness: P@5, R@10, MRR, hallucination rate, citation coverage, latency p95.

#### 4.1.3 Тренажёр (`/api/trainer/*` + `/api/srs/*`)
- Квиз с автоматической генерацией из корпуса.
- **SRS** (Spaced Repetition System): алгоритм SM-2, flip-карточки, оценки 0–5, due-list, `from-route` (создать карточки прямо из маршрута Куратора).
- UI: Тренажёр-таб → секция «Интервальные повторения».

#### 4.1.4 Пушкинская карта (`/api/pushkin/*`)
- Каталог 50+ событий × все федеральные округа РФ.
- Фильтры: регион, тема (литература / театр / музей / кино), возраст 14–22, дата, цена.
- UI-таб «Пушкинская карта».

#### 4.1.5 152-ФЗ контур (`/api/privacy/*`)
- `consent` (согласие на обработку ПДн).
- `export` (выгрузка собственных данных по запросу).
- `forget` (удаление учётных данных).
- `audit_log` — журнал событий безопасности (отказы, ошибки, массовые операции).

#### 4.1.6 Safety & guardrails (`/api/safety/*`)
- Jailbreak detection (классификатор атак на промпт).
- PII redaction (e-mail, телефон, ИНН).
- Prompt-injection защита.
- Журнал отказов (`/api/safety/refusals`).

#### 4.1.7 Observability (`/metrics` + JSON-логи)
- Prometheus exposition: RPS, p95-latency, LLM cost ($ / 1k токенов), hallucination rate, citation coverage.
- JSON-логи (structured logging).
- OpenTelemetry traces (готово к подключению Jaeger / Yandex Cloud Tracing).

#### 4.1.8 Frontend / PWA / WCAG
- PWA: устанавливается как приложение, работает offline (precache shell + cached corpus).
- Иконки 192/512 (SVG).
- WCAG 2.1: skip-link, role=tablist/tab/tabpanel, aria-selected/-controls, roving tabindex, focus-visible:ring, aria-hidden на декоративных иконках.
- 8 табов: Куратор / Тренажёр / Поиск / Интеллект-карта / Пушкинская карта / Дашборд / Privacy / Audit.

#### 4.1.9 Telegram-бот
- aiogram 3.x.
- 4 команды + 4 сценария (запрос маршрута, квиз, инфо, помощь).
- Использует тот же backend через httpx.

## 7. Соответствие нормативке РФ

### 7.1 152-ФЗ «О персональных данных»

- Локализация в РФ (Yandex Cloud, mosX-east1).
- Согласие на обработку (`/api/privacy/consent`).
- Право на доступ (`/api/privacy/export`).
- Право на удаление (`/api/privacy/forget`).
- Минимизация сбора: для гостя — никаких ПДн; для зарегистрированного — только e-mail / Telegram-id.
- Журнал ПДн (`05_Юр_пакет/06_реестр_ПДн.md`).
- Юр. структура: ООО + регистрация в Роскомнадзоре как оператор ПДн (в roadmap, см. F7).

### 7.2 ФГОС / ФИПИ / нацпроекты

- Соответствие приказам Минпросвещения 287/2021 и 732/2022 (ФГОС ООО / СОО).
- Соответствие спецификациям ФИПИ для ЕГЭ/ОГЭ по литературе и истории (ФИПИ-карточки в `06_Методички/`).
- Указ Президента РФ № 490 от 10.10.2019 «О развитии ИИ в РФ» — соответствует принципам прозрачности, безопасности, недискриминации.
- Нацпроект «Культура» — прямая интеграция с Пушкинской картой.
- Нацпроект «Образование» — закрытие KPI «доля школьников, использующих цифровые образовательные сервисы».

### 7.3 Авторское право

- Корпус — только public-domain (произведения, у которых истёк срок действия исключительного права).
- Источники: РГБ (открытая коллекция), НЭБ (открытая коллекция).
- Каждая цитата сопровождается ссылкой на конкретное издание и страницу.
- Никаких пиратских PDF / нелегальных копий.

### 7.4 Этический протокол

См. `05_Юр_пакет/08_этический_протокол.md`. Шесть принципов:
1. **Прозрачность** — каждый ответ показывает источник и ссылку.
2. **Достоверность** — при отсутствии источника модель отвечает «не знаю», без галлюцинаций.
3. **Защита данных** — 152-ФЗ, минимизация сбора, локализация в РФ.
4. **Недискриминация** — ровный сервис вне зависимости от региона / школы / дохода семьи.
5. **Защита от вреда** — фильтры на содержимое, возрастные режимы 14+ и 18+.
6. **Открытый стек** — документированные модели, версии промптов, журнал инцидентов.
