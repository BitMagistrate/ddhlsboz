## Что отличит ЧитАИ от конкурентов

1. **Полностью российский LLM-стек.** GigaChat MAX + YandexGPT 5 Pro — единственный сильный AI-проект в EdTech без OpenAI/Anthropic. Это аргумент №1 для Минкульта/Минпросвещения.
2. **Public-domain корпус только.** РГБ + НЭБ + проект «Гутенберг (RU)». Никаких авторских прав 1973+. Это убирает риск, который убил бы B2G-контракт с Минпросвещения.
3. **Honest RAG benchmark.** Открытый репозиторий с golden set, скриптом прогонки, метриками. Это уровень, которого нет у zachet.app, GigaChat-ассистента и большинства образовательных AI.
4. **Trust-stack.** Цитаты гарантированно НЕ из LLM. 6 категорий отказа с audit-логом. 152-ФЗ контур (consent + export + forget).
5. **Анти-zachet позиционирование.** «Учим читать и понимать, а не писать ИИ-сочинения за тебя». Это ВЕРБАЛЬНО важно для жюри, которое смотрит на этическую сторону AI в образовании.
6. **Пушкинская карта.** 50+ событий по регионам с фильтром. Это пересекается с программой Минкультуры — сильный B2G-аргумент.

### C-1. Голосовой режим через Алису — «Алиса, спроси у ЧитАИ, кто такой Печорин» — ★★★★★

**Почему wow:** ЧитАИ становится доступен через Алису-колонку дома. Школьник готовится к ЕГЭ — спрашивает голосом — получает ответ с цитатами. Это абсолютный буст для номинации «Цифровая среда» и для пресс-релиза.

**Что добавить:**
- Навык в Yandex Dialogs (бесплатно).
- Webhook → существующий `/api/curator/route` с `output=voice`.
- Адаптация ответов под голос: 200 слов max, без markdown, без длинных URL.
- Сохранение istorii в Алисе (multi-turn).

**Цена:** 1.5–2 дня. **Δ балла:** +0.8.
**В видео:** колонка Яндекса на столе → школьник спрашивает → Алиса голосом цитирует «Героя нашего времени».
**Бонус:** доступность для слабовидящих — отдельный аргумент в заявке.

### C-2. «Спроси Раскольникова» — ролевой режим (RAG-карточка героя) — ★★★★★

**Почему wow:** пользователь выбирает героя (Раскольников / Базаров / Чацкий / Татьяна Ларина / Печорин) → задаёт ему вопросы → герой отвечает только цитатами и парафразами из произведения. **Подростки от этого фанатеют.**

**Что добавить:**
- Новая страница `/character/<slug>` во фронте.
- Backend: расширить RAG поверх корпуса 1 произведения, system prompt «Отвечай как герой X, только опираясь на текст произведения. Если ответа нет в тексте — скажи: "В романе об этом не сказано"».
- 5 героев × 1 произведение каждый = 5 готовых карточек.
- Audit-лог: какие цитаты использовал в ответе.

**Цена:** 1.5 дня. **Δ балла:** +0.7.
**В видео:** диалог с Раскольниковым «Зачем ты убил старуху?» → герой отвечает цитатами из главы 5. Эпично и моментально вирально.
**Безопасность:** RAG over single book → невозможно сгенерировать что-то вне канона. Защищаемо для жюри.

### C-4. AI-разбор сочинения через фото тетради (Yandex Vision) — ★★★★

**Почему wow:** школьник пишет сочинение → фоткает тетрадь → ЧитАИ распознаёт текст → разбирает: что хорошо, что слабо, нужны ли ещё цитаты, какова логика. **Это убийца zachet.app**, потому что:
- zachet.app пишет сочинение **за** ученика → запрещено.
- ЧитАИ **разбирает уже написанное** учеником → этично, разрешено в школе.

**Что добавить:**
- Endpoint `/api/curator/essay-review` принимает изображение / PDF.
- Yandex Vision OCR → текст → промпт «оцени по критериям ЕГЭ К1–К12».
- Фронт: страничка «Загрузи фото своего сочинения».
- Audit: сохраняем только критерии оценки, текст не сохраняем (152-ФЗ).

**Цена:** 1.5 дня. **Δ балла:** +0.7.
**В видео:** фото мятого тетрадного листа → AI говорит «К1 — 1/1, К2 — 0/3 (нет цитат), К4 — 2/3 (логика хромает в 3 абзаце)». Тут жюри видит реальный продукт.
**Это становится №1 продающим сценарием для родителей.**
