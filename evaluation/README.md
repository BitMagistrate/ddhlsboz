# Открытый RAG-бенчмарк ЧитАИ

Каталог содержит **полностью открытый** отчёт о точности RAG-поиска
ЧитАИ по public-domain корпусу русской литературы. Все цифры
воспроизводимы через `poetry run python -m app.benchmark` из
[`backend/`](../backend/).

## Что измеряем

| Метрика     | Что считает                                                 | Где применима           |
| ----------- | ----------------------------------------------------------- | ----------------------- |
| **MRR**     | средний обратный ранг правильного ответа                    | первая релевантная книга |
| **Recall@5**| доля запросов, у которых эталон попал в топ-5               | покрытие                 |
| **P@5**     | precision: доля релевантных в топ-5                         | плотность ответа         |
| **P@3**     | precision@3 — для коротких ответов в боте                   | мобильный UX             |
| **Hit@5**   | хотя бы один релевантный документ среди топ-5               | sanity-check             |

## Свежий отчёт

Файл `report.json` обновляется автоматически из CI на каждый PR в
`main`. См. [benchmark.json в артефактах CI](../.github/workflows/ci.yml).

```json
{
  "metrics": {
    "p_at_5": 0.275,
    "p_at_3": 0.4236,
    "recall_at_5": 0.9854,
    "mrr": 0.9792,
    "hit_at_5": 1.0,
    "n_queries": 48,
    "misses": []
  }
}
```

> CI-гейт: **MRR ≥ 0.85**, **Recall@5 ≥ 0.90**. Регрессия — красный билд.

## Почему мы блокируем по MRR, а не по P@5

В наших задачах у запроса обычно один «правильный» документ
(например, «Преступление и наказание» для вопроса о Раскольникове).
P@5 при росте корпуса механически снижается: в топ-5 попадает 4
других похожих, релевантных, но не эталонных. MRR показывает
ровно ту метрику, которая важна пользователю — нашли ли мы
именно нужное произведение первым.

## Воспроизводимость

```bash
cd backend
poetry install --no-root --with dev
poetry run python -m app.benchmark | tee evaluation/report.json
```

Корпус полностью открыт ([`backend/app/corpus.py`](../backend/app/corpus.py)),
вопросы тоже ([`backend/app/benchmark.py`](../backend/app/benchmark.py)) —
любой регион/университет может прогнать тот же набор и сравнить с
любой другой RAG-системой. PR с улучшениями приветствуются.
